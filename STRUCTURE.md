## Notebook Execution Order

Notebooks are numbered and should be run sequentially. Notebook 00 produces the
dataset that all others consume.

```
00_data_pipeline ──► data/processed/pokemon_clean.csv
                     data/raw/flavor_text.csv
        │
        ├──► 01_eda_and_profiling
        ├──► 02_classification
        ├──► 03_regression
        ├──► 04_clustering
        ├──► 05_time_series
        ├──► 06_nlp_text_analysis (also reads flavor_text.csv)
        │
        └──► 07_model_comparison (references models from 02 + 03)
             └──► 08_storytelling_dashboard (synthesizes all findings)
```

## Datathon Skill Mapping

Each notebook targets a specific skill area tested in UIUC Datathon competitions:

| Notebook | Datathon Skill | Real-World Analog |
|----------|---------------|-------------------|
| 00 | Data acquisition & feature engineering | First 30-60 min of any datathon |
| 01 | EDA & data profiling | Understanding the dataset before modeling |
| 02 | Classification | Credit risk (2025), churn prediction |
| 03 | Regression | Spending forecasting (2025), economic prediction (2018) |
| 04 | Clustering & segmentation | Customer segments, risk tiers |
| 05 | Time series & trends | Stock prediction (2019), economic impact (2018) |
| 06 | NLP & text analysis | Chatbot satisfaction (2023), call transcripts (2024) |
| 07 | Model comparison & evaluation | Justifying model choice to judges |
| 08 | Storytelling & recommendations | Presentation that wins the competition |

## Pokemon-to-Business Domain Mapping

| Pokemon Concept | Business Analog |
|----------------|-----------------|
| Individual Pokemon | Customer / account |
| Base stats (HP, Atk, Def, ...) | Behavioral features / spending patterns |
| Types (Fire, Water, ...) | Customer segments |
| Legendary / Mythical status | High-value / premium tier customers |
| Generations (1-9) | Time periods / fiscal quarters |
| Pokedex flavor text | Customer reviews / support transcripts |
| Capture rate | Acquisition difficulty / conversion rate |
| Evolution chains | Customer lifecycle / account upgrades |
| Stat total (BST) | Overall customer value score |

## Notebook Template

Every notebook should follow this structure:

```
1. Title & Datathon Skill
   - What skill this practices
   - How it maps to a real datathon problem

2. Setup
   - Imports
   - Load data from data/processed/

3. Analysis / Modeling
   - Clear section headers with markdown explanations
   - Code with outputs and visualizations

4. Key Findings
   - 3-5 bullet points summarizing results

5. Business Recommendations
   - Translate findings into actionable insights
   - Practice the language judges want to hear
```
